# 注释与文档

## P.CMT.01 文档应该围绕 What 和 How 为核心来构建

### 【描述】 

文档应该始终围绕两个方向来构建：

1. What : 用于阐述代码为什么而构建。
2. how : 用于阐述代码如何去使用。

## P.CMT.02 文档应该保持简短精干

### 【描述】

1. 文档内容用语应该尽量简短精干，不宜篇幅过长。
2. 使用通俗易懂的描述而尽量避免使用专业技术术语。
